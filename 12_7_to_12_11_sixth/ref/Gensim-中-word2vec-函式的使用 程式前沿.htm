模型建立

Gensim 中 word2vec 模型的輸入是經過分詞的句子列表，即是某個二位陣列。但由於中文沒有像英文那麼自帶天然的分詞，所有這裡我們簡單採用 jieba 來進行分詞處理。

# 引入 word2vec
from gensim.models import word2vec
# 引入日誌配置
import logging
logging.basicConfig(format='%(asctime)s : %(levelname)s : %(message)s', level=logging.INFO)
# 引入分詞工具
import jieba
# 引入資料集
raw_sentences = ["但由於中文沒有像英文那麼自帶天然的分詞","所以我們第一步採用分詞"]
# 切分詞彙
sentences = []
for s in raw_sentences:
tmp = []
for item in jieba.cut(s):
tmp.append(item)
sentences.append(tmp)
# print(sentences)
# 構建模型
model = word2vec.Word2Vec(sentences, min_count=1)
# 進行詞向量輸出
model['中文']

但這種方法在資料集比較大的情況下會佔用大量的 RAM，Gensim 本身只是要求能夠迭代的有序句子列表，因此在工程實踐中我們可以使用一些自定義的生成器，只在記憶體中儲存單條語句，並且在匯入記憶體之前進行分詞操作。

Word2Vec 引數

  • sentences: 可以是一個 list，對於大型的預料集，我們建議使用 BrownCorpus，Text8Corpus，LineSentence 來進行構建。
from gensim.models.word2vec import LineSentence
from gensim.models.word2vec import Text8Corpus
sentences = LineSentence('a.txt')
sentences = Text8Corpus('a.txt')
  • size: 詞向量的維度。
  • alpha: 模型初始的學習率。
  • window: 表示在一個句子中，當前詞於預測詞在一個句子中的最大距離。
  • min_count: 用於過濾操作，詞頻少於 min_count 次數的單詞會被丟棄掉，預設值為 5。
  • max_vocab_size: 設定詞向量構建期間的 RAM 限制。如果所有的獨立單詞數超過這個限定詞，那麼就刪除掉其中詞頻最低的那個。根據統計，每一千萬個單詞大概需要 1GB 的RAM。如果我們把該值設定為 None ，則沒有限制。
  • sample: 高頻詞彙的隨機降取樣的配置閾值，預設為 1e-3，範圍是 (0, 1e-5)。
  • seed: 用於隨機數發生器。與詞向量的初始化有關。
  • workers: 控制訓練的並行數量。
  • min_alpha: 隨著訓練進行，alpha 線性下降到 min_alpha。
  • sg: 用於設定訓練演算法。當 sg=0，使用 CBOW 演算法來進行訓練；當 sg=1，使用 skip-gram 演算法來進行訓練。
  • hs: 如果設定為 1 ，那麼系統會採用 hierarchica softmax 技巧。如果設定為 0（預設情況），則系統會採用 negative samping 技巧。
  • negative: 如果這個值大於 0，那麼 negative samping 會被使用。該值表示 “noise words” 的數量，一般這個值是 5 – 20，預設是 5。如果這個值設定為 0，那麼 negative samping 沒有使用。
  • cbow_mean: 如果這個值設定為 0，那麼就採用上下文詞向量的總和。如果這個值設定為 1 （預設情況下），那麼我們就採用均值。但這個值只有在使用 CBOW 的時候才起作用。
  • hashfxn: hash函式用來初始化權重，預設情況下使用 Python 自帶的 hash 函式。
  • iter: 演算法迭代次數，預設為 5。
  • trim_rule: 用於設定詞彙表的整理規則，用來指定哪些詞需要被剔除，哪些詞需要保留。預設情況下，如果 word count < min_count，那麼該詞被剔除。這個引數也可以被設定為 None，這種情況下 min_count 會被使用。
  • sorted_vocab: 如果這個值設定為 1（預設情況下），則在分配 word index 的時候會先對單詞基於頻率降序排序。
  • batch_words: 每次批處理給執行緒傳遞的單詞的數量，預設是 10000。

記憶體

word2vec 引數模型儲存在 NumPy 矩陣中。矩陣中的每一行是單詞的向量，這個向量用浮點數（單精度，也稱為 4 位元組）表示。

假 設三個這樣的矩陣同時儲存在 RAM 中（如果是矩陣正在工作中，那麼該數量應該減少到兩個，甚至一個）。因為，如果你輸入的單詞量是 100000 個，詞向量你要求長度為 200 ，那麼模型所佔的記憶體為 100000 * 200 * 4 * 3 =~ 229 MB。

大型語料庫

在真實的訓練環境中，我們面對的都是比較大型的語料庫，在這裡我們以 word2vec 官方的 text8 為例子，具體如下：

sentences = word2vec.Text8Corpus('text8')
model = word2vec.Word2Vec(sentences, size=10)

我們在前面也提到了，如果是對大量的輸入語聊集或者需要整合磁碟上多個資料夾中的資料，那麼我們可以使用迭代器的方式來讀取資料，而不是一次性的將全部內容讀取到記憶體中，從而來節省 RAM 空間，具體如下：

class MySentences(object):
def __init__(self, dirname):
self.dirname = dirname
def __iter__(self):
for fname in os.listdir(self.dirname):
for line in open(os.path.join(self.dirname, fname)):
yield line.split()
sentences = MySentences('/some/directory') # a memory-friendly iterator
model = gensim.models.Word2Vec(sentences)
